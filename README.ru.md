# Componenta IP Address

Объект значения для IPv4/IPv6 с валидацией, нормализацией, классификацией, проверками CIDR, маскированием, бинарным преобразованием и сериализацией в JSON или строку.

Используйте его для метаданных запроса, правил доступа, audit log и инфраструктурного кода, где не стоит передавать сырые IP-строки.

## Установка

```bash
composer require componenta/ip-address
```

## Связанные пакеты

Пакет самодостаточный и не требует соседних библиотек для валидации IP.

| Пакет | Зачем может использоваться рядом |
|---|---|
| `componenta/http-trusted-proxy-middleware` | Промежуточный обработчик доверенного прокси может передавать IP клиента в атрибуты запроса. |
| `componenta/policy` | Политики доступа могут использовать IP или CIDR-диапазоны как часть контекста. |
| `componenta/validation` | Может валидировать сырой пользовательский ввод до создания `IpAddress`. |

## Использование

```php
use Componenta\Stdlib\IpAddress;

$ip = IpAddress::fromString('192.168.1.10');

(string) $ip;              // "192.168.1.10"
$ip->version->name;        // "V4"
$ip->isPrivate();          // true
$ip->isPublic();           // false
$ip->isInRange('192.168.1.0/24'); // true
$ip->masked();             // "192.168.1.***"
```

## Конструкторы и валидация

- `fromString()` валидирует и нормализует или выбрасывает `InvalidArgumentException`
- `tryFromString()` возвращает `null` для некорректного ввода
- `isValid()`, `isValidV4()` и `isValidV6()` выполняют статические проверки

Нормализация использует `inet_pton()`/`inet_ntop()`.

## Классификация

Объект умеет определять:

- IPv4 vs IPv6
- RFC 1918/private ranges и IPv6 unique local range
- loopback
- link-local
- multicast
- публичную маршрутизируемость

`isPublic()` не является обратным значением `isPrivate()`. Зарезервированные, документационные, loopback, multicast и link-local диапазоны могут быть ни публичными, ни приватными.

## CIDR и преобразование

`isInRange()` возвращает `false` для некорректного CIDR, некорректного префикса или несовпадения версии IP.

`toBinary()` возвращает упакованную бинарную форму. `toLong()` возвращает числовую строку и требует расширение GMP для безопасного преобразования IPv6.

## Сравнение

- `equals()` сравнивает нормализованные значения и не считает IPv4 и IPv4-mapped IPv6 равными
- `equalsCanonical()` сначала сопоставляет IPv4 с IPv6 и может сравнивать адреса разных версий
